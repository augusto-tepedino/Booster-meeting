# Regras do Projeto — Booster Meeting / Test

## Aprovação de Alterações

Antes de aplicar **qualquer alteração em código** (criação, edição ou exclusão de arquivos), sempre exiba as mudanças propostas ao usuário — seja via plano de implementação, diff ou descrição detalhada — e aguarde aprovação explícita antes de prosseguir.

Isso se aplica a:

- Edições em arquivos de teste (`.spec.ts`, `.test.ts`)
- Edições em arquivos de configuração (`playwright.config.ts`, `package.json`, etc.)
- Criação de novos arquivos
- Qualquer refatoração ou reorganização de código
